Introduction

Les logical properties servent à rendre le code plus léger en le rendant plus logique. Elles permettent aux éléments de suivre le sens de lecture assigné par la langue utilisée.

Pour avoir ce résultat, on va écrire : 

                
    .avatar {
        margin-right: 1rem;
    }

    html[dir="rtl"] .avatar {
        margin-right: 0;
        margin-left: 1rem;
    }

A l'aide des logical properties , on peut mettre : 

                
    .avatar {
        margin-inline-end: 1rem;
    }

Les logical properties influent sur le : positionnement margin padding sizing border-radius...

Leur intérêt est de réduire les lignes de code et de faire un code adaptable aux différentes langues.

À savoir : flex et grid intègrent déjà les logical properties !

Manipuler les sens de lecture

Pour utiliser les logical properties, il faut quand même modifier le sens de lecture des textes. Pour ce faire, vous avez l'attribut dir pour gauche/droite et la propriété css writing-mode pour haut/bas : 

                
 <p dir="rtl">Sens de lecture</p>
            
                
  .element{
    writing-mode: vertical-rl;
  }

<p dir="ltr">

<p dir="rtl">

Vous avez la référence ? C'est flexbox froggy :))
.element{ writing-mode: vertical-rl; };

Et voilà ! Les logical properties ne sont pas un principe dur à comprendre et à apprendre. Leur implémentation est une très bonne chose puisqu'elles allègent (à leurs échelles...) le code mais sont aussi plus pratiques et logiques pour les développeurs; il n'y a aucune raison de ne pas les adopter ;).
